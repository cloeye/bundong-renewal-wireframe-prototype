# 번동제일교회 리뉴얼 와이어프레임 UI/UX 규칙

## 1. 전체 성격

이 프로토타입은 최종 디자인 시안이 아니라, 정보 구조와 화면 흐름을 검증하기 위한 그레이스케일 와이어프레임이다.

실제 사진, 강한 장식, 과도한 브랜드 컬러보다 레이아웃, 위계, 여백, 콘텐츠 흐름이 먼저 보여야 한다.

화면은 교회 내부자만을 위한 구성이 아니라, 처음 방문한 사람과 고령 사용자도 쉽게 이해할 수 있는 구조를 우선한다.

## 2. 색상 규칙

전체 색상은 그레이스케일을 기본으로 한다.

섹션 구분은 색상 장식이 아니라 흰색과 연회색 배경 교차로 만든다.

섹션 배경은 콘텐츠 폭에만 갇히지 않고 화면 전체 너비로 이어져야 한다.

강한 검정 배경이나 진한 강조 박스는 특별한 목적이 있을 때만 사용한다. 기본 콘텐츠 페이지에서는 사용하지 않는다.

## 3. 섹션 구조 규칙

각 페이지는 섹션 단위로 명확히 나뉜다.

메인페이지처럼 `SEC-01`, `SEC-02` 형태의 작은 섹션 라벨을 사용해 와이어프레임 구조를 식별할 수 있게 한다.

섹션 라벨은 작고 조용해야 하며, 실제 콘텐츠보다 눈에 띄면 안 된다.

1차 메뉴 하위페이지의 첫 영역은 `SEC-01`이면서 서브타이틀 역할을 함께 한다.

하위 페이지의 `SEC-01`은 `우리교회는`에서 사용한 구조를 기준으로 한다. `처음오셨나요`의 첫 하위 메뉴 `어서오세요`와 `예배안내`도 같은 구조를 따른다.

- 상단에는 `SEC-01` 라벨만 작게 둔다
- 이어서 큰 타이틀과 작은 설명문
- 필요하면 배경 또는 대표 이미지 성격의 사진 영역을 함께 사용한다
- 카드가 따로 떠 보이지 않고 하나의 섹션처럼 보여야 한다

`SEC-01`이 서브타이틀 역할을 하는 페이지는 별도의 큰 상단 히어로를 중복으로 두지 않는다. 1차 메뉴 아래의 2차 메뉴 탭 다음에 `SEC-01`이 본문 첫 영역으로 바로 이어지게 한다.

`SEC-01`에는 버튼형 섹션 이동 네비게이터를 넣지 않는다. 첫 화면은 메뉴보다 타이틀처럼 보여야 한다.

`SEC-02` 이후 일반 콘텐츠 섹션은 `SEC 라벨 + 타이틀 + 짧은 설명`을 먼저 보여주고, 그 아래에 버튼형 섹션 이동 네비게이터를 둔다.

버튼형 섹션 이동 네비게이터는 현재 섹션 안에서 이동을 돕는 장치다. 섹션의 의미를 설명하는 역할은 타이틀과 설명문이 담당한다.

버튼형 섹션 이동 네비게이터의 첫 버튼은 공식 2차 메뉴명이므로 나머지 섹션 버튼과 시각적으로 구분한다. 첫 버튼은 부모 항목처럼 보이게 더 강한 대비를 주고, 나머지 버튼은 그 아래 부속 섹션처럼 가볍게 둔다.

버튼형 섹션 이동 네비게이터의 첫 버튼은 `SEC-01`로 돌아가는 역할을 한다. 예를 들어 `우리교회는` 버튼을 누르면 페이지 상단의 `SEC-01` 영역으로 스크롤된다.

`처음오셨나요`처럼 하위 섹션이 여러 개인 페이지도 모든 본문 섹션에 같은 버튼형 섹션 이동 네비게이터를 반복한다. `어서오세요`, `담임목사 인사말`, `새가족 등록안내`, `새가족 교육`, `새가족 섬김이`, `새가족환영회`, `금주의 새가족` 모두 같은 이동 체계를 사용한다.

섹션 내부의 보조 정보나 필터는 버튼형 섹션 이동 네비게이터와 같은 형태를 반복하지 않는다. 교육 일정, 장소 정보는 하단 메타 정보로 정리하고, 연도 필터와 해시태그 필터는 구분선과 텍스트형 인덱스로 표현한다.

메인페이지도 같은 섹션 문법을 따른다. 메인은 방문자의 첫 화면이므로 콘텐츠 성격은 더 요약적일 수 있지만, 섹션 라벨, 타이틀, 설명, 카드, 필터, 버튼의 시각 언어는 `번동제일교회는`과 `처음오셨나요` 하위 콘텐츠 기준에서 벗어나지 않는다.

필요하지 않은 부연설명은 넣지 않는다.

## 4. 섹션 간 여백 규칙

섹션과 섹션 사이가 끊어진 카드 모음처럼 보이면 안 된다.

메인페이지처럼 화면 전체 폭의 배경이 이어지고, 내부 콘텐츠만 일정한 최대 폭 안에 정렬되어야 한다.

섹션 내부 여백은 충분히 주되, 콘텐츠가 지나치게 공중에 떠 있거나 과장된 랜딩페이지처럼 보이면 안 된다.

메인페이지의 각 섹션도 하위 페이지와 같은 세로 호흡을 사용한다. 특별한 이벤트 페이지가 아닌 이상 짧은 띠 배너, 강한 어두운 박스, 과한 히어로 장식으로 리듬을 깨지 않는다.

## 5. 타이포그래피 규칙

제목은 짧고 직관적이어야 한다.

설명형 문장보다 영역의 의미가 바로 보이는 말을 우선한다.

좋은 예:

- `교회 소식`
- `주보와 순서지`
- `첫 걸음이, 낯설지 않도록`
- `우리교회 핵심가치`
- `하나님이 이끄신 시간`

피해야 할 예:

- `자주 찾는 메뉴를 빠르게 확인할 수 있는 영역`
- `시대별 흐름 안에서 중요한 사건을 차분하게 읽을 수 있도록 정리합니다`

보조문구는 꼭 필요할 때만 사용한다.

사용할 경우 설명서가 아니라 콘텐츠의 감정과 맥락을 보완하는 한 줄이어야 한다.

첫 영역의 설명은 기능 설명보다 교회다운 환대와 관계를 담는 문장으로 작성한다.

좋은 예:

- `우리교회는 ㅣ 핵심가치 ㅣ 환영의 말씀 ㅣ 초대합니다`
- `함께 행복을 만들어가는 번동제일교회`
- `대한예수교장로회(통합) 소속으로, 하나님께서 주시는 참된 행복을 함께 나누는 공동체입니다.`
- `어서오세요`
- `어서오세요 환영하고 축복합니다`
- `번동제일교회는 오직 주님 안에 거하며, 함께 행복한 교회를 만들어 나가는 공동체입니다.`

대표 이미지가 필요한 페이지는 `SEC-01` 안에서 배경 이미지 또는 대표 이미지 플레이스홀더로 처리한다.

`우리교회는`, `걸어온 길`, `시설안내`, `교회조직`, `섬기는 사람들`, `오시는 길`의 `SEC-01` 타이틀과 설명은 같은 폰트 굵기, 행간, 자간을 사용한다. 페이지마다 다른 컴포넌트를 쓰더라도 첫 영역의 타이포 리듬은 동일해야 한다.

메인페이지의 섹션 타이틀도 같은 타이포 리듬을 사용한다. 메인이라고 해서 별도의 얇은 제목, 과장된 큰 제목, 컬러 강조 제목을 만들지 않는다.

서브타이틀 역할을 겸하는 `SEC-01` 타이틀과 설명은 아래 문구를 기준으로 한다.

| 메뉴 | 타이틀 | 설명 |
|---|---|---|
| 우리교회는 | 함께 행복을 만들어가는 번동제일교회 | 대한예수교장로회(통합) 소속으로, 하나님께서 주시는 참된 행복을 함께 나누는 공동체입니다 |
| 걸어온 길 | 한 천막에서 시작된 예배가 오늘의 교회가 되기까지 | 1964년부터 오늘까지, 번동제일교회가 같은 자리에서 걸어온 믿음의 시간입니다 |
| 시설안내 | 예배와 모임이 이루어지는 공간입니다 | 예배와 모임이 이루어지는 주요 공간은 본관과 (선교)교육관으로 나누어져 있습니다 |
| 교회조직 | 교회의 구조와 공동체를 한눈에 | 예배, 교육, 찬양, 교구까지 번동제일교회를 이루는 공동체를 소개합니다 |
| 섬기는 사람들 | 함께 교회를 섬기는 사람들입니다 | 담임목사님을 비롯해 번동제일교회를 함께 섬기는 교역자와 직분자를 소개합니다 |
| 오시는 길 | 언제든지 오세요, 기다리고 있습니다 | 대중교통과 자가용 이용 방법, 주차 안내까지 찾아오시는 길을 안내합니다 |

`교회조직` 페이지의 본문 섹션 타이틀과 설명은 아래 문구를 기준으로 한다.

| 섹션 | 타이틀 | 설명 |
|---|---|---|
| SEC-02 교회조직 | 교회 운영 구조 | 교회를 이끄는 기관입니다 |
| SEC-03 교육부 | 교육부 소개 | 다음 세대를 말씀으로 세웁니다 |
| SEC-04 청년부 | 청년부 소개 | 청년들이 함께 예배하고 교제합니다 |

| SEC-07 교구소개 | 교구 소개 | 가까운 이웃과 함께하는 신앙 공동체입니다 |

`예배안내` 페이지의 본문 섹션 타이틀과 설명은 아래 문구를 기준으로 한다.

| 섹션 | 타이틀 | 설명 |
|---|---|---|
| SEC-01 예배안내 | 예배가 삶의 중심이 되는 교회입니다 | 하나님께 드리는 예배가 날마다 이어지는 공동체입니다 |
| SEC-02 예배시간/장소 | 예배 시간과 장소 안내 | 모든 예배 일정을 확인하세요 |
| SEC-03 교회학교 예배 | 교회학교 예배 안내 | 각 부서별 예배 시간을 확인하세요 |
| SEC-04 차량운행안내 | 차량 운행 안내 | 주일 운행 노선과 시간표를 확인하세요 |

`말씀&찬양` 페이지는 예배 영상의 종류를 사용자가 바로 고를 수 있도록 아래 구조를 기준으로 한다.

| SEC-04 찬양예배 | 찬양예배 | 찬양으로 함께 드리는 주일 오후 예배입니다 |
| SEC-05 기도회 | 기도회 | 말씀과 기도로 한 주간을 준비하는 시간입니다 |

| SEC-07 집회 및 행사 | 집회 및 행사 | 특별한 은혜와 기쁨의 시간들입니다 |

`말씀&찬양` 안의 세부 분류는 버튼처럼 보이지 않게 처리한다. `기도회`, `찬양대`, `집회 및 행사`의 하위 분류는 위아래 라인이 있는 텍스트형 인덱스로 표현한다.

## 6. UX 라이팅 규칙

영어 표기는 불필요하면 제거한다.

`Our Church`, `Core Value`, `History`, `Vision`, `Archive` 같은 장식적 영어는 사용하지 않는다.

교회 홈페이지의 주요 사용자는 다양한 연령대이므로, 메뉴명과 섹션명은 한국어 중심으로 직관적으로 작성한다.

문구는 과장하지 않고, 교회다운 따뜻함과 공공성을 유지한다.

문장 톤은 다음 순서를 우선한다.

- 짧다
- 이해하기 쉽다
- 따뜻하다
- 설명보다 안내에 가깝다
- 내부자 용어를 최소화한다

## 7. 메뉴 UX 규칙

하위 메뉴는 왼쪽 사이드 메뉴보다 상단 가로 탭 방식이 더 적합하다.

고령 사용자도 현재 위치와 주변 메뉴를 한눈에 볼 수 있어야 한다.

가로 탭은 라인맵 스타일을 사용한다.

- 메뉴는 한 줄로 배치
- 현재 메뉴는 하단 라인 또는 굵기 차이로 표시
- 과한 색상 사용 금지
- 그레이스케일 유지
- 메뉴 사이에는 얇은 구분선 사용 가능

드롭다운보다 직접 보이는 가로 탭이 우선이다.

특히 `번동제일교회는`, `처음오셨나요`, `예배안내`처럼 여러 하위 콘텐츠를 가진 메뉴는 가로 탭이 더 직관적이다.

## 8. 콘텐츠 페이지 규칙

하위 페이지는 여러 개의 끊어진 페이지보다, 필요한 경우 하나의 긴 랜딩페이지처럼 구성한다.

예를 들어 `예배안내` 안에 `예배시간/장소`와 `차량운행안내`를 한 페이지에 배치하고, 탭 클릭 시 해당 섹션으로 스크롤 이동한다.

`번동제일교회는`도 1차 메뉴 안에서 `우리교회는`, `걸어온 길`, `시설안내`, `교회조직`, `섬기는 사람들`, `오시는 길`이 같은 UI 규칙을 공유해야 한다.

## 9. 카드와 박스 규칙

박스는 정보를 정리하기 위한 장치이지 장식이 아니다.

불필요한 2중 박스는 제거한다.

카드가 너무 많으면 페이지가 무거워지므로, 타임라인처럼 연속성이 중요한 콘텐츠는 카드 반복보다 라인과 리스트 중심으로 정리한다.

카드가 필요한 경우:

- 핵심가치 3개
- 사람 소개
- 예배 시간 정보
- 연결 배너
- 갤러리 썸네일
- 메인페이지의 요약 카드

카드가 과하면 안 되는 경우:

- 역사 연표
- 긴 목회 메시지
- 본문 중심 소개글

카드는 기본적으로 흰 배경, 얇은 선, 8px 이하의 반경을 사용한다. 그림자, 입체감, 강한 색상 활성 상태는 쓰지 않는다.

필터나 카테고리 선택 영역은 버튼처럼 보이지 않게 한다. `처음오셨나요`의 연도별 보기와 금주의 새가족 태그처럼 위아래 라인과 텍스트 인덱스로 정리한다.

## 10. 사진 영역 규칙

이 프로토타입은 그레이스케일 와이어프레임이므로 실제 사진을 넣지 않는다.

사진이 필요한 위치는 `사진 영역`, `예배 모습 사진`, `역사 대표 사진 영역`처럼 자리만 잡는다.

사진 영역은 콘텐츠 구조상 필요한 곳에만 둔다.

역사 페이지처럼 반복 사진이 페이지를 무겁게 만들면 제거하고, 대표 사진 1개 또는 주요 구간에만 배치한다.

## 11. 역사 콘텐츠 규칙

`걸어온 길`은 자료 아카이브가 아니라 읽히는 이야기여야 한다.

연도별 박스를 반복하지 않는다.

시대별 흐름 안에 연도와 사건을 리스트로 배치한다.

시대명은 짧고 의미 있게 정리한다.

- 시작의 시간
- 함께 세운 교회
- 이어진 사명
- 새로운 출발

각 시대는 `시대명 + 짧은 맥락 + 연도 리스트` 구조를 사용한다.

## 12. 특별 페이지 규칙

`창립 60주년`, `故 백낙기 목사 기념관` 같은 특별 페이지는 일반 페이지보다 감정적 밀도가 높을 수 있다.

하지만 기본 디자인 원칙은 동일하게 유지한다.

특별 페이지도 지나치게 이벤트 페이지처럼 튀지 않도록, 그레이스케일, 절제된 여백, 읽기 쉬운 구조를 유지한다.

## 13. 접근성 및 사용성 규칙

고령 사용자도 읽기 쉬워야 한다.

- 메뉴는 숨기지 말고 가능한 직접 노출한다
- 클릭 가능한 항목은 명확히 보이게 한다
- 글자는 너무 작게 쓰지 않는다
- 긴 문단은 폭을 제한하고 행간을 넉넉히 둔다
- 버튼과 탭은 충분한 높이와 여백을 가진다
- 현재 위치 표시가 분명해야 한다

## 14. 최종 판단 기준

화면을 볼 때 다음 질문에 모두 "예"라고 답할 수 있어야 한다.

- 지금 내가 어느 메뉴에 있는지 바로 아는가?
- 이 섹션이 어떤 역할인지 바로 보이는가?
- 설명문이 너무 많지 않은가?
- 섹션 간 배경과 여백이 일정한가?
- 메인페이지, 우리교회는, 걸어온 길이 같은 사이트처럼 보이는가?
- 고령 사용자도 메뉴와 흐름을 이해할 수 있는가?
- 실제 사진이 없어도 구조가 충분히 검증되는가?

## 핵심 요약

그레이스케일로 절제하고, 한국어로 쉽게 말하고, 섹션 구조는 규칙적으로, 메뉴는 보이게, 설명은 줄이고, 사용자가 다음 행동을 바로 알게 한다.
